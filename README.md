The Two Pillars of SyncThe system manages two distinct documents that can exist at either the Global (User) level or the Local (Project) level.DocumentFocusGlobal Level ContentProject Level ContentCLAUDE.mdThe User[User context, general tech stack].Specific repo goals, local dependencies, project-specific task history.SOUL.mdThe PersonaGeneral communication style, core values (wit, brevity), and broad engagement rules.Domain-specific "vibe" (e.g., more academic for OMSCS, more pragmatic for home DIY).2. The Sync LifecycleStep 0: Gather (The Ingestion)The tool acts as a "data harvester" for your Claude export files.Input: users.json, memories.json, projects.json, and conversations.json.Analysis: An LLM-backed parser scans for Identity Markers (facts for CLAUDE.md) and Behavioral Markers (patterns for SOUL.md).Conflict Detection: Identifying when a new conversation contradicts a stored memory (e.g., a move from one project team to another).Step 1: Initial Onboarding & ReviewThe "Founding" phase where the user establishes the baseline.The Fact Review: Agent Sync presents a categorized list of "Proposed Truths."Example: "I've noticed you frequently mention Kotlin Coroutines in the Photos repo. Should I add this to the local CLAUDE.md?"Persona Calibration: The tool generates a draft SOUL.md based on how the agent has successfully interacted with you in the past.Action: You "bless" or "tweak" the tone, ensuring it matches your preference for a supportive, grounded, and witty peer.Step 2: Regular Review (The Pulse)A recurring maintenance loop to keep the files from going stale.Delta Detection: Every time you provide a new export, the tool highlights only what has changed.Context Decay: The tool flags facts that haven't been mentioned in X months (e.g., a completed home renovation) and asks if they should be moved to an ARCHIVE.md.Learning Extraction: "In your last 5 sessions, you corrected me 3 times on brevity. Updating SOUL.md to prioritize conciseness."3.
